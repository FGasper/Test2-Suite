Event Facets

Event Facets

The current event system is limited. New behaviors require writing new event types, and branching based on event type.

Facets to the rescue!

  • Facets are information and/or behaviors that can be attached to events
  • All current behaviors have a facet
  • All current event information is included in facets
  • The 'Trace' object is now a facet
  • Any event can have any facets
Optimal Pass

Optimized event for passing assertions

New optimized 'Pass' event, and context methods.
  • One call to send the event and release the context
  • Returns true
  • Path through the Test2 internals is shortcutted and optimized

Yath

Test2::Harness (Yet Another Test Harness)

This is a modern alternative to prove.
  • preload+fork option
  • recursive by default
  • recieves events instead of parsing TAP.
  • can parse TAP. if needed (works with really old tests)
  • Able to output alternatives to TAP.
There is a prototype on CPAN, but it is undergoing a complete rewrite.
